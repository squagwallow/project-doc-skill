## Retrofit notes

In Docs, tables are readable but inert — operator cannot filter or sort rows, just read them top-to-bottom. For *reference* tables like milestones and rate ladders this is fine. For *working* data like flagged jobs, this would be a regression; flagged-jobs correctly lives in a Sheet.

Operator can hyperlink from this Doc to specific Sheet ranges, but native cross-referencing is one of Docs' weakest features.
